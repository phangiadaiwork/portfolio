NestJS Next.js 16 Go PostgreSQL RabbitMQ Redis Docker Prisma 7 BullMQ MinIO

A comprehensive multi-tenant, event-driven SaaS platform built on microservices architecture. Designed to handle real-world e-commerce operations with enterprise-grade identity management, intelligent notifications, and blockchain integration.

🏗️ Architecture Overview

The system follows a strict microservices architecture with each service owning its own database (Database-per-Service pattern). Communication is primarily event-driven through RabbitMQ, with synchronous REST calls only when immediate response is required.

  • @saas/event-contracts — TypeScript interfaces for all RabbitMQ events across services
  • @saas/permission-scanner — Auto-scans @DefinePermission decorators and syncs via RabbitMQ
  • @saas/permission-core — @RequirePermissions decorator + guard for authorization

🧠 Architecture Principles

  • Database-per-Service — No cross-DB access. Data shared only through events.
  • Event-First — Async via RabbitMQ; synchronous calls only when mandatory.
  • Token is Contract — JWT carries all auth context. No service-to-service auth calls.
  • Zero-Trust — Every request authenticated and authorized.
  • Eventual Consistency — Services are eventually consistent across boundaries.
  • Soft Delete — deletedAt pattern with partial unique indexes for data integrity.
